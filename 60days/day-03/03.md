# node 核心 fs、path、os

路径处理是服务端最基础的操作之一。不要去手动拼接路径问题，使用 path 模块来处理路径

```js
import path from 'node:path';


// 获取路径各部分
const filePath = '/home/user/documents/report.pdf';
path.dirname(filePath);   // '/home/user/documents'
path.basename(filePath);  // 'report.pdf'
path.basename(filePath, '.pdf'); // 'report'
path.extname(filePath);   // '.pdf'
```

`fs` 模块提供三套 API：

| API 风格 | 示例 | 特点 |
|---------|------|------|
| 同步 | `fs.readFileSync()` | 阻塞主线程，简单脚本用 |
| 回调 | `fs.readFile()` | 传统异步，不推荐新代码使用 |
| Promise | `fs.promises.readFile()` | ✅ **推荐**，配合 async/await |

writeFile 写入文件内容(覆盖写入)
appendFile 追加写入文件内容（不覆盖写入）
fs.stat 获取文件信息 （同步）
fs.rename 
fs.rm or fs.unlink 删除文件

检查文件是否存在使用同步方案 `fs.existsSync()`  
异步存在竞态问题，建议使用同步方案
```js
const exists = fs.existsSync(filePath);
```
